39 class ABCInterfaceClass(InterfaceClass):
40 """
41 An interface that is automatically derived from a
42 :class:`abc.ABCMeta` type.
43  
44 Internal use only.
45  
46 The body of the interface definition *must* define
47 a property ``abc`` that is the ABC to base the interface on.
48  
49 If ``abc`` is *not* in the interface definition, a regular
50 interface will be defined instead (but ``extra_classes`` is still
51 respected).
52  
53 Use the ``@optional`` decorator on method definitions if
54 the ABC defines methods that are not actually required in all cases
55 because the Python language has multiple ways to implement a protocol.
56 For example, the ``iter()`` protocol can be implemented with
57 ``__iter__`` or the pair ``__len__`` and ``__getitem__``.
58  
59 When created, any existing classes that are registered to conform
60 to the ABC are declared to implement this interface. This is *not*
61 automatically updated as the ABC registry changes. If the body of the
62 interface definition defines ``extra_classes``, it should be a
63 tuple giving additional classes to declare implement the interface.
64  
65 Note that this is not fully symmetric. For example, it is usually
66 the case that a subclass relationship carries the interface
67 declarations over::
68  
69 >>> from zope.interface import Interface
70 >>> class I1(Interface):
71 ... pass
72 ...
73 >>> from zope.interface import implementer
74 >>> @implementer(I1)
75 ... class Root(object):
76 ... pass
77 ...
78 >>> class Child(Root):
79 ... pass
80 ...
81 >>> child = Child()
82 >>> isinstance(child, Root)
83 True
84 >>> from zope.interface import providedBy
85 >>> list(providedBy(child))
86 [<InterfaceClass __main__.I1>]
87  
88 However, that's not the case with ABCs and ABC interfaces. Just
89 because ``isinstance(A(), AnABC)`` and ``isinstance(B(), AnABC)``
90 are both true, that doesn't mean there's any class hierarchy
91 relationship between ``A`` and ``B``, or between either of them
92 and ``AnABC``. Thus, if ``AnABC`` implemented ``IAnABC``, it would
93 not follow that either ``A`` or ``B`` implements ``IAnABC`` (nor
94 their instances provide it)::
95  
96 >>> class SizedClass(object):
97 ... def __len__(self): return 1
98 ...
99 >>> from collections.abc import Sized
100 >>> isinstance(SizedClass(), Sized)
101 True
102 >>> from zope.interface import classImplements
103 >>> classImplements(Sized, I1)
104 None
105 >>> list(providedBy(SizedClass()))
106 []
107  
108 Thus, to avoid conflicting assumptions, ABCs should not be
109 declared to implement their parallel ABC interface. Only concrete
110 classes specifically registered with the ABC should be declared to
111 do so.
112  
113 .. versionadded:: 5.0.0
114 """
